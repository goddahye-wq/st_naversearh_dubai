# [작업지시서] Naver API 트렌드 분석 Streamlit 대시보드 구축

본 문서는 수집된 네이버 API 데이터를 기반으로 사용자가 키워드를 입력하여 트렌드를 비교하고, 기초 EDA(탐색적 데이터 분석)를 수행할 수 있는 대화형 대시보드 구축 계획을 수립합니다.

## 1. 프로젝트 개요
- **목적**: 키워드별 검색 추이 비교 및 쇼핑/블로그 데이터의 다각도 분석
- **도구**: 
    - **Framework**: Streamlit
    - **Visualization**: Plotly (Interactive Charts)
    - **Data Handling**: Pandas

## 2. 대시보드 레이아웃 및 메뉴 구성
사용자 편의성을 위해 사이드바 메뉴와 메인 페이지의 탭 구조를 활용합니다.

- **Sidebar**:
    - 키워드 입력 필드 (예: 두바이 초콜릿, 두바이 쫀득쿠키)
    - 데이터 수집 기간 설정
    - 분석 대상 데이터 소스 선택 (트렌드, 블로그, 쇼핑)
- **Main Tabs**:
    1. **Trend Analysis**: 키워드 간 검색량/쇼핑 트렌드 비교
    2. **Shopping EDA**: 쇼핑 상품 데이터의 가격 및 통계 분석
    3. **Blog Insights**: 블로그 게시물 텍스트 및 트렌드 분석
    4. **Raw Data**: 수집 데이터 전체 보기

## 3. 구현 항목 세부 사항

### 3.1 Plotly 기반 시각화 (그래프 5종 이상)
1. **시계열 라인 차트**: 기간별 키워드 검색량 추이 비교 (Datalab Search)
2. **가격 분포 히스토그램**: 쇼핑 상품의 가격대별 빈도 분석
3. **카테고리 비율 파이 차트**: 쇼핑 검색 결과 내 카테고리 비중 확인
4. **쇼핑몰별 바 차트**: 주요 판매처(Mall)별 상품 등록 수 비교
5. **가격 vs 평점 산점도**: 상품 가격과 고객 반응 간의 상관관계 시각화

### 3.2 데이터 요약 및 분석 (표 5종 이상)
1. **기초 통계량 요약 테이블**: 평균 가격, 최고/최저가, 상품 수 등 요약
2. **Top 10 상품 리스트**: 리뷰 수 또는 관련도 높은 상품 목록
3. **카테고리별 요약 피벗 테이블**: 카테고리별 평균 가격 및 상품 수 합계
4. **최근 블로그 피드 테이블**: 키워드와 관련된 최신 뉴스/블로그 요약
5. **키워드 비교 요약 테이블**: 키워드 A와 B의 전체 수치 간결한 비교

## 4. 기술적 로직 계획
1. **Data Loading**: 
    - `raw_data/` 폴더의 CSV 파일들을 자동으로 감지하여 로드
    - 파일명의 주차/날짜 정보를 활용한 데이터 관리
2. **Keyword Filtering**:
    - 사용자 입력에 따라 DataFrame을 실시간으로 Slice 처리
3. **Interactive Components**:
    - Plotly의 `hover`, `zoom` 기능을 활성화하여 상세 데이터 확인 가능하게 구성
4. **Styling**:
    - `st.set_page_config`를 통한 대시보드 레이아웃(wide) 및 테마 설정

## 5. 최종 결과물

- 가독성 높은 인터랙티브 EDA 결과물
